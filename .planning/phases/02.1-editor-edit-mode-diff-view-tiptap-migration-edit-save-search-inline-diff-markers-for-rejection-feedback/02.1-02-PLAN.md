**1. Create TipTapEditor component** at `packages/frontend/src/components/docs/TipTapEditor.tsx`:

```typescript
import { useEditor, EditorContent } from '@tiptap/react';
import StarterKit from '@tiptap/starter-kit';
import { Markdown } from '@tiptap/markdown';
import TaskList from '@tiptap/extension-task-list';
import TaskItem from '@tiptap/extension-task-item';
import Placeholder from '@tiptap/extension-placeholder';
import CodeBlockLowlight from '@tiptap/extension-code-block-lowlight';
import { common, createLowlight } from 'lowlight';
import { useEffect, useRef, useCallback } from 'react';
```

Props interface:
```typescript
interface TipTapEditorProps {
  content: string;          // markdown string
  editable: boolean;        // read-only vs edit mode
  onUpdate?: (markdown: string) => void;  // called on every edit with serialized markdown
}
```

Initialize the lowlight instance at module scope: `const lowlight = createLowlight(common);`

Use `useEditor` with these extensions:
- `StarterKit.configure({ codeBlock: false })` (replaced by CodeBlockLowlight)
- `Markdown` (official @tiptap/markdown)
- `TaskList`
- `TaskItem.configure({ nested: true })`
- `Placeholder.configure({ placeholder: 'Empty document' })`
- `CodeBlockLowlight.configure({ lowlight })`

Editor options:
- `content`: the content prop
- `contentType: 'markdown'` (CRITICAL -- without this, markdown renders as raw text)
- `editable`: the editable prop
- `shouldRerenderOnTransaction: false` (TipTap 3 React performance)
- `onUpdate: ({ editor }) => { if (onUpdate) onUpdate(editor.getMarkdown()); }`

**Scroll preservation on content update:**
- Use a `useRef` for the container div ref
- When content prop changes and editable is false (read-only mode live-reload), capture `containerRef.current.scrollTop` before calling `editor.commands.setContent(content, { contentType: 'markdown' })`, then restore scrollTop after.
- Use a `useEffect` watching `[content]` that: (a) checks `editor` exists and is NOT destroyed, (b) skips if editable is true and isDirty (don't overwrite user edits -- handled by conflict warning in Plan 05), (c) captures scrollTop, (d) calls `editor.commands.setContent(content, { contentType: 'markdown' })`, (e) restores scrollTop via `requestAnimationFrame`.
- IMPORTANT: Do NOT re-create the editor when content changes. Use `setContent()` to preserve undo history and cursor.

**Editable toggle:**
- Use a `useEffect` watching `[editable, editor]` that calls `editor.setEditable(editable)` when the prop changes. Do NOT rely on the useEditor `editable` option for runtime reactivity (see RESEARCH.md anti-pattern).

**Expose editor instance** via a callback prop `onEditorReady?: (editor: Editor) => void` -- called in a useEffect when editor is first available. This allows parent components (DocsView) to access the editor for save/search operations.

Render:
```tsx
return (
  <div ref={containerRef} className="flex-1 overflow-y-auto">
    <div className="prose prose-sm prose-zinc max-w-none px-6 py-6">
      <EditorContent editor={editor} />
    </div>
  </div>
);
```

Implement the new actions:
```typescript
setTabMode: (tabId, mode) => {
  set((state) => ({
    tabs: state.tabs.map((t) =>
      t.id === tabId ? { ...t, mode } : t
    ),
  }));
},

setTabDirty: (tabId, isDirty) => {
  set((state) => ({
    tabs: state.tabs.map((t) =>
      t.id === tabId ? { ...t, isDirty } : t
    ),
  }));
},

snapshotTabs: new Map(),

captureSnapshot: () => {
  const state = get();
  const snapshot = new Map<string, string>();
  for (const tab of state.tabs) {
    snapshot.set(tab.filePath, tab.content);
  }
  set({ snapshotTabs: snapshot });
},

getSnapshot: (filePath) => {
  return get().snapshotTabs.get(filePath);
},

clearSnapshot: (filePath) => {
  set((state) => {
    if (filePath) {
      const newMap = new Map(state.snapshotTabs);
      newMap.delete(filePath);
      return { snapshotTabs: newMap };
    }
    return { snapshotTabs: new Map() };
  });
},
```

Also update `updateTabContent` to reset isDirty when content comes from live-reload (external update):
```typescript
updateTabContent: (tabId, content) => {
  set((state) => ({
    tabs: state.tabs.map((t) =>
      t.id === tabId
        ? { ...t, content, lastLoadedAt: Date.now(), isDirty: false }
        : t
    ),
  }));
},
```

Note: `snapshotTabs` should NOT be persisted to localStorage. Keep it out of the `partialize` function (which currently only persists `activeView`).

```typescript
import { useAppStore } from '../store/index.js';
import { TabBar } from '../components/docs/TabBar.js';
import { TipTapEditor } from '../components/docs/TipTapEditor.js';
import { EditorToolbar } from '../components/docs/EditorToolbar.js';
import { EmptyDocs } from '../components/docs/EmptyDocs.js';
import { useCallback, useRef } from 'react';
import type { Editor } from '@tiptap/react';
```

Wire up:
- Read `tabs`, `activeTabId`, `setActiveTab`, `closeTab`, `setTabMode`, `setTabDirty` from store
- Get `activeTab` from tabs array
- Keep a ref for the editor instance: `const editorRef = useRef<Editor | null>(null);`
- `handleEditorReady` callback sets `editorRef.current = editor`
- `handleToggleMode` callback:
  - If currently read mode, switch to edit mode: `setTabMode(activeTab.id, 'edit')`
  - If currently edit mode, switch to read mode: `setTabMode(activeTab.id, 'read')` and `setTabDirty(activeTab.id, false)` (simple toggle -- conflict/unsaved prompts added in Plan 05)
- `handleUpdate` callback: when editor content changes during edit, call `setTabDirty(activeTab.id, true)` and update the tab's content in the store for later saving

Layout (vertical flex):
```tsx
<div className="flex-1 flex flex-col bg-white">
  <TabBar
    tabs={tabs.map((t) => ({ id: t.id, displayName: t.displayName, isDirty: t.isDirty }))}
    activeTabId={activeTabId}
    onTabClick={setActiveTab}
    onTabClose={closeTab}
  />
  {activeTab ? (
    <>
      <EditorToolbar
        mode={activeTab.mode}
        isDirty={activeTab.isDirty}
        onToggleMode={handleToggleMode}
      />
      <TipTapEditor
        key={activeTab.id}
        content={activeTab.content}
        editable={activeTab.mode === 'edit'}
        onUpdate={handleUpdate}
        onEditorReady={handleEditorReady}
      />
    </>
  ) : (
    <EmptyDocs />
  )}
</div>
```

**IMPORTANT about the `key` prop on TipTapEditor:** Use `activeTab.id` as the key so that switching tabs creates a fresh editor instance for each tab. This avoids stale ProseMirror state when switching between documents.
