**3. Add conflict warning banner to DocsView** in `packages/frontend/src/views/DocsView.tsx`:

Read conflict state from store:
```typescript
const conflictFilePath = useAppStore((s) => s.conflictFilePath);
```

Add two conflict resolution handlers:
```typescript
const handleReloadFromDisk = useCallback(() => {
  if (!activeTab || !conflictContent) return;
  // Accept disk version: update content, reset dirty, switch to read mode
  updateTabContent(activeTab.id, conflictContent);
  setTabDirty(activeTab.id, false);
  setTabMode(activeTab.id, 'read');
  clearConflict();
}, [activeTab, conflictContent, updateTabContent, setTabDirty, setTabMode, clearConflict]);

const handleKeepMyChanges = useCallback(() => {
  // Dismiss the warning, keep editing
  clearConflict();
}, [clearConflict]);
```

Render conflict banner ONLY when conflictFilePath matches activeTab.filePath:
```tsx
{conflictFilePath && activeTab && conflictFilePath === activeTab.filePath && (

    <div className="ml-auto flex items-center gap-2">
```

**4. Add unsaved changes prompt to mode toggle** in `packages/frontend/src/views/DocsView.tsx`:

Update `handleToggleMode`:
```typescript
const handleToggleMode = useCallback(() => {
  if (!activeTab) return;

  if (activeTab.mode === 'edit' && activeTab.isDirty) {
    // Prompt: save or discard before switching to read mode
    const shouldSave = window.confirm(
      'You have unsaved changes. Click OK to save before switching to read-only, or Cancel to discard changes.'
    );
    if (shouldSave) {
      // Save first, then switch mode
      handleSave().then(() => {
        setTabMode(activeTab.id, 'read');
      });
      return;
    } else {
      // Discard: reset content to last saved version by re-fetching from store
      // The store content was last set by updateTabContent (from live-reload or save)
      // Just reset dirty and switch mode
      setTabDirty(activeTab.id, false);
    }
  }

  const newMode = activeTab.mode === 'read' ? 'edit' : 'read';
  setTabMode(activeTab.id, newMode);
}, [activeTab, handleSave, setTabMode, setTabDirty]);
```

NOTE: `handleSave` must be declared before `handleToggleMode` in the component body (or both wrapped in useCallback and handleSave included in handleToggleMode's deps).

Also make handleSave return a Promise so it can be awaited:
```typescript
const handleSave = useCallback(async () => { ... });
```
