<decisions>
## Implementation Decisions

### TipTap Migration Strategy
- **Full TipTap migration**: Replace react-markdown MarkdownViewer with a TipTap editor using StarterKit + tiptap-markdown + CodeBlockLowlight + TaskList + TaskItem + Placeholder
- TipTap editor defaults to `editable: false` (read-only), toggle via EditorToolbar
- Existing MarkdownViewer.tsx will be replaced entirely (not kept alongside TipTap)
- Scroll preservation must be re-implemented for TipTap (ProseMirror scroll API differs from DOM scrollTop)

### Diff View is a Must-Have
- Diff view (US7) is required for v1 — not optional
- Depends on TipTap being in place (ProseMirror Decoration API)
- Uses `diff` library's diffLines() for line-level comparison
- Snapshot captured on chat submit, compared when filesystem:changed fires after agent response

### Backend Addition: POST /api/files/save
- New endpoint needed for edit mode save
- Same path-traversal validation as GET /api/files/read
- Must suppress file watcher triggering for self-initiated saves (to avoid reload loop)

### Claude's Discretion
- Search implementation approach (TipTap search extension vs ProseMirror find-text)
- DiffOverlay decoration styling (green/red backgrounds, dismiss mechanism)
- Conflict warning UI (banner vs modal dialog)
- Tab overflow scroll indicator style

</decisions>

<specifics>
## Specific Ideas

- tiptap-markdown for bidirectional markdown serialization (load .md → ProseMirror → edit → serialize back to .md)
- Existing Zustand store already has setTabMode and setTabDirty actions from Phase 2 — wire them
- File watcher suppression: backend can use a short debounce window after save, or client can set a flag to ignore the next filesystem:changed event
- Undo/redo is built into TipTap StarterKit's History extension — no separate implementation needed

</specifics>
