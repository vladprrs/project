### Human Verification Required

#### 1. Diff Decorations After Tab Switch (Re-confirmation of UAT issue #10 fix)

**Test:** Open a document in Docs view. Send any chat message (captures snapshot). Externally modify the file on disk (e.g., `echo "# Added line" >> specs/.../spec.md`). Switch back to Docs view if needed.
**Expected:** The green "Document changed" banner appears AND the added content paragraph/heading has a green left-border highlight (`.diff-added` CSS class applied to the ProseMirror node). No extra browser-console errors.
**Why human:** Timing-sensitive behavior: decorations must survive React's `useEffect` + `requestAnimationFrame` cycle when switching views. The fix (commit 2724ca4) targets this exact race condition. Only a live browser can confirm the fix holds.

#### 2. Unsaved Changes Prompt on Mode Toggle

**Test:** Open a document, toggle to edit mode, type any text (dirty dot appears). Click the mode toggle button ("Editing" → "Read-only").
**Expected:** A browser `confirm()` dialog appears: "You have unsaved changes. Click OK to save before switching to read-only, or Cancel to discard changes." Clicking OK saves the file and switches to read mode. Clicking Cancel discards edits and switches to read mode.
**Why human:** Browser dialog interaction requires a running browser; cannot be invoked programmatically in a headless check.

#### 3. Conflict Warning Banner on External File Change

**Test:** Open a document in edit mode, type unsaved changes (dirty dot visible). In a terminal, modify the file on disk. Switch back to the browser tab.
**Expected:** An amber banner appears: "File changed on disk. This document was modified externally while you have unsaved changes." with "Reload from disk" and "Keep my changes" buttons. Clicking "Reload from disk" replaces editor content with disk version and clears the dirty state. Clicking "Keep my changes" dismisses the banner without changing the editor content.
**Why human:** Requires live coordination between filesystem chokidar events, WebSocket delivery, and React state rendering.
