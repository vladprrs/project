# Phase 02.1 Plan 03: Save + Search Functionality Summary

**Save button (Ctrl+S) posting to /api/files/save and prosemirror-search wrapped as SearchExtension with SearchBar overlay**

## Accomplishments
- EditorToolbar extended with Save button: visible in edit mode, disabled (text-zinc-400 "Saved") when clean, active (text-blue-600 "Save") when dirty
- DocsView handleSave fetches POST /api/files/save, calls updateTabContent + setTabDirty(false) on success
- Ctrl+S/Cmd+S global keyboard shortcut registered via window.addEventListener (works regardless of editor focus)
- SearchExtension created as a TipTap Extension wrapping prosemirror-search plugin, exposing setSearchQuery and clearSearch commands
- SearchBar component: auto-focuses on mount, updates search query on input change, next/prev via findNext/findPrev, close clears search
- Cmd+F/Ctrl+F opens SearchBar; Escape closes it
- SearchExtension added to TipTapEditor extensions array
- All TypeScript compiles cleanly (0 errors)

## Decisions Made
- `updateTabContent` called after successful save to persist markdown content AND reset isDirty — this is the correct use of updateTabContent (live-reload writes from backend, save writes from frontend, both go through same path)
- Keyboard shortcuts registered on window (not TipTap shortcuts) so Ctrl+S works even when editor is not focused
- SearchBar positioned `absolute top-2 right-4` within a `relative` container that wraps TipTapEditor — clean overlay without layout shift
