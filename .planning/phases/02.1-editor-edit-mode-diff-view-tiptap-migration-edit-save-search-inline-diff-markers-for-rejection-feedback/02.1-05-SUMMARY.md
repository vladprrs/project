## Decisions Made
- Conflict state stored in Zustand (not component-local) so useWebSocket hook can set it and DocsView can render it -- cross-component communication requires shared store
- Used window.confirm for unsaved changes prompt -- native, simple, zero-dependency approach appropriate for a developer tool
- Converted SearchBar to controlled component with query owned by DocsView parent to enable automatic search refresh when content updates during live-reload
