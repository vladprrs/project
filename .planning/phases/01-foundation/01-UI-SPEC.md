# Phase 1 -- UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom Tailwind components) |
| Icon library | lucide-react 1.0.1 |
| Font | System font stack: `ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` (Tailwind default `font-sans`) |

**Rationale:** Phase 1 has three UI elements (icon rail, connection dot, placeholder views). No component library overhead is justified. Tailwind 4 with CSS-first configuration provides all styling. shadcn not initialized because the project uses raw Tailwind with custom components and "Custom theme system" is explicitly out of scope per REQUIREMENTS.md.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, connection dot margin |
| sm | 8px | Nav button padding, gap between rail items |
| md | 16px | Content area padding |
| lg | 24px | Not used in Phase 1 |
| xl | 32px | Not used in Phase 1 |
| 2xl | 48px | Icon rail width (w-12 = 48px) |
| 3xl | 64px | Not used in Phase 1 |

Exceptions: none

**Phase 1 specific usage:**
- Icon rail width: 48px (Tailwind `w-12`) -- provides 44px effective touch target for icon buttons with 8px padding
- Icon button hit area: 36px (20px icon + 8px padding each side) -- meets 36px minimum for mouse-primary desktop interface
- Gap between nav icon buttons: 4px (Tailwind `gap-1`)
- Rail top padding: 8px (Tailwind `pt-2`)
- Rail bottom padding: 12px (Tailwind `pb-3`)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |

**Phase 1 usage:**
- Placeholder view text ("Chat", "Docs", "Kanban"): Heading role -- 20px, semibold 600, used centered in the content area per D-07
- Tooltip text: Label role -- 14px, regular 400
- Connection status tooltip text: Label role -- 14px, regular 400
- Body role declared for forward compatibility but not used in Phase 1 placeholder views

**Weight rationale:** Two weights only -- `font-normal` (400) and `font-semibold` (600) in Tailwind. Label uses regular weight because tooltip text (the only Label usage in Phase 1) does not require a mid-weight distinction; the smaller 14px size alone differentiates it from Body.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` / Tailwind `white` | Content area background, icon rail background |
| Secondary (30%) | `#F4F4F5` / Tailwind `zinc-100` | Active nav icon button background |
| Accent (10%) | `#18181B` / Tailwind `zinc-900` | Active nav icon color only |
| Destructive | `#EF4444` / Tailwind `red-500` | Disconnected status dot only |

**Additional semantic colors (Phase 1 only):**

| Color | Value | Usage |
|-------|-------|-------|
| Border | `#E4E4E7` / Tailwind `zinc-200` | Rail-to-content vertical divider (D-02) |
| Muted text | `#A1A1AA` / Tailwind `zinc-400` | Inactive nav icon default color |
| Muted text hover | `#52525B` / Tailwind `zinc-600` | Inactive nav icon hover color |
| Placeholder text | `#A1A1AA` / Tailwind `zinc-400` | Centered view name text in placeholder views |
| Connected | `#22C55E` / Tailwind `green-500` | WebSocket connected status dot (D-06) |
| Reconnecting | `#F59E0B` / Tailwind `amber-500` | WebSocket reconnecting status dot (D-06) |
| Disconnected | `#EF4444` / Tailwind `red-500` | WebSocket disconnected status dot (D-06) |

Accent reserved for: active nav icon button text color (`zinc-900`) -- indicates the currently selected view. Never applied to hover states, connection dots, or placeholder text.

**Color rationale:** The zinc palette provides a neutral, professional tone consistent with developer tooling (VS Code activity bar reference from D-01). Same background color for rail and content area per D-02, differentiated only by the `zinc-200` border.

## Interaction States

### Nav Button States

| State | Visual |
|-------|--------|
| Default (inactive) | `zinc-400` icon, transparent background |
| Hover (inactive) | `zinc-600` icon, transparent background |
| Active (selected view) | `zinc-900` icon, `zinc-100` background |
| Hover (active) | No change from active state (already visually prominent) |
| Focus-visible | Browser default focus ring (outline) for keyboard accessibility |

### Connection Dot States

| State | Visual | Tooltip |
|-------|--------|---------|
| Connected | `green-500` solid circle | "Connected" |
| Reconnecting | `amber-500` solid circle | "Reconnecting..." |
| Disconnected | `red-500` solid circle | "Disconnected" |

No animation on the connection dot. State changes are immediate (color swap). Animation is deferred to later phases if desired.

### View Switching

| Trigger | Behavior |
|---------|----------|
| Click nav button | Immediate view swap, no transition animation |
| Page load | Restore last active view from localStorage (UX-02) |
| First visit (no stored state) | Default to "Chat" view |
