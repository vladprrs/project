## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 (21px) | Chat messages, document prose body text |
| Label | 12px | 500 (medium) | 1.33 (16px) | Tab names, timestamps, status text, "Load more" link |
| Heading | 18px | 600 (semibold) | 1.33 (24px) | Empty state headings, view section titles |
| Mono | 13px | 400 (regular) | 1.5 (19.5px) | Code blocks in chat messages and document viewer |

Tailwind classes:
- Body: `text-sm font-normal leading-relaxed` (14px / 400 / 1.5)
- Label: `text-xs font-medium leading-tight` (12px / 500 / 1.33)
- Heading: `text-lg font-semibold leading-snug` (18px / 600 / 1.33)
- Mono: `font-mono text-[13px] leading-relaxed`

Document viewer prose typography is handled by `@tailwindcss/typography` plugin with `prose prose-sm prose-zinc` classes. The plugin provides its own heading scale (h1-h6), list styles, table styles, and code block styles within the rendered markdown. No custom overrides needed.

## Interaction Contracts

### Chat Input Behavior

| Trigger | Action | Visual Feedback |
|---------|--------|-----------------|
| User types text + presses Enter (or clicks Send) | `sendMessage()` called, input cleared | Input disabled, Send button shows spinner or disabled state |
| User presses Shift+Enter | Newline inserted in input (no submit) | Input grows up to 4 lines max, then scrolls internally |
| Stream starts (`status: 'submitted'`) | Activity indicator appears below message list | Pulsing blue dot + "Thinking..." text |
| Stream chunks arrive (`status: 'streaming'`) | Assistant message text appends progressively | Indicator text changes to "Generating response...", message text grows |
| Stream completes (`status: 'ready'`) | Activity indicator removed, input re-enabled | Input focus restored |
| Stream error (`status: 'error'`) | Error banner appears above input | Red text with error message + "Retry" button |
| User clicks Retry on error | `regenerate()` called | Error banner dismissed, streaming restarts |

### Chat Auto-Scroll

| Condition | Behavior |
|-----------|----------|
| User is scrolled to bottom (within 100px of bottom) | Auto-scroll on every new chunk/message |
| User has scrolled up (more than 100px from bottom) | Do NOT auto-scroll; show a "scroll to bottom" button at bottom-right of message area |
| User clicks "scroll to bottom" | Smooth scroll to bottom, button disappears |
| New user message sent | Always scroll to bottom regardless of position |

### Chat Message History Loading

| Trigger | Action | Visual Feedback |
|---------|--------|-----------------|
| View mounts with active feature | Fetch last 50 messages from `GET /api/chat/messages` | Skeleton or empty state until loaded |
| User scrolls to top + `hasMore` is true | Show "Load older messages" link at top | Link text, not a button |
| User clicks "Load older messages" | Fetch next page with cursor, prepend to list | "Loading..." text replaces link, scroll position preserved after prepend |
| Active feature changes | Clear messages, fetch new feature's history | Brief empty state during fetch |

### Artifact Link Behavior

| Trigger | Action | Visual Feedback |
|---------|--------|-----------------|
| Chat message contains artifact filename (e.g. `spec.md`, `plan.md`) | Filename rendered as blue underlined link | `text-blue-600 hover:underline cursor-pointer font-medium` |
| User clicks artifact link | Switch `activeView` to `docs`, open tab for that file, set as active tab | View switches, tab appears in tab bar, document content loads |
| File already open in a tab | Switch to existing tab (no duplicate) | Active tab highlight moves |

### Document Tab Behavior

| Trigger | Action | Visual Feedback |
|---------|--------|-----------------|
| Tab opened (via artifact link or auto-open) | Tab added to tab bar, file content fetched via `GET /api/files/read`, set as active | New tab appears with display name, content renders in viewer |
| User clicks a tab | `setActiveTab()` called | Border moves to clicked tab, content switches |
| User clicks close (x) on tab | Tab removed. If it was active, adjacent tab becomes active (prefer right, then left). If last tab, empty state shows. | Tab disappears, content switches or empty state appears |
| WebSocket `filesystem:changed` for open file | Tab content updated silently | Scroll position preserved (capture before update, restore after render) |
| WebSocket `filesystem:created` for file in active feature dir | Auto-open new tab | Tab appears, does NOT steal focus from current active tab |
| WebSocket `filesystem:deleted` for open file | Tab closed automatically | Tab removed, adjacent tab activated |

### Tab Display Names

| File Path | Display Name |
|-----------|-------------|
| `specs/{feature}/spec.md` | `spec.md` |
| `specs/{feature}/plan.md` | `plan.md` |
| `specs/{feature}/tasks.md` | `tasks.md` |
| `specs/{feature}/research.md` | `research.md` |
| `specs/{feature}/contracts/api.md` | `contracts/api.md` |
| Other paths | Last two path segments (e.g. `foo/bar.md`) |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Send" (chat input submit button label) |
| Chat empty state heading | "Start a conversation" |
| Chat empty state body | "Type a command to invoke a workflow stage. Try asking to generate a spec or build a plan." |
| Chat empty state (no feature) heading | "No feature selected" |
| Chat empty state (no feature) body | "Activate a feature to start chatting. Features are loaded from the specs/ directory." |
| Docs empty state heading | "No documents open" |
| Docs empty state body | "Open a document from a chat artifact link, or documents will appear here automatically as agents create them." |
| Chat error state | "Something went wrong: {error.message}. Retry" (where "Retry" is a clickable action) |
| Chat streaming: submitted | "Thinking..." |
| Chat streaming: streaming | "Generating response..." |
| Load more link | "Load older messages" |
| Load more loading | "Loading..." |

| Tab close button | "x" character, tooltip: "Close {displayName}" |

No destructive actions exist in Phase 2. Chat history is append-only (no delete). Tab close is non-destructive (content is on disk, not in the tab).
