Add `onFinish` callback to the `useChat` configuration. The `onFinish` callback receives the completed message. Extract the text content from its `parts` array and persist it. Use `activeFeatureRef.current` instead of `activeFeature` directly:

In the `useChat({...})` config object, add after the `onError` callback:

```typescript
onFinish: (message) => {
  // Persist completed assistant message per D-08
  // Use activeFeatureRef.current to avoid stale closure if user
  // switched features during streaming
  const textContent = message.parts
    ?.filter((p: any) => p.type === 'text' && p.text)
    .map((p: any) => p.text)
    .join('') ?? '';
  if (textContent && activeFeatureRef.current?.id) {
    fetch('/api/chat/messages', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        featureId: activeFeatureRef.current.id,
        role: 'assistant',
        content: textContent,
      }),
    }).catch((err) => console.error('[chat] Failed to persist assistant message:', err));
  }
},
```

This ensures that:
- User messages are persisted immediately on send (already in ChatView.handleSend from Plan 03)
- Assistant messages are persisted when streaming completes via onFinish
- Per D-08, only user and assistant roles are persisted (no system events)
- Feature ID is read from a ref, not a closure, preventing stale feature assignment
